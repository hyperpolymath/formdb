== Version Numbering

FormDB follows https://semver.org/[Semantic Versioning]:

* `MAJOR.MINOR.PATCH` (e.g., `1.2.3`)
* Pre-1.0: Breaking changes may occur in any release
* Post-1.0: Breaking changes only in major versions

== Stability Tiers (Post-1.0)

=== Stable
* FQL query syntax
* Core data types
* Provenance model

=== Experimental
* Dependent types (FQLdt)
* Self-normalizing features
* Distributed consensus

=== Internal
* Block layout details
* Journal encoding
* ABI internals

== Breaking Change Policy

.Pre-1.0 (Current)
* Breaking changes documented in CHANGELOG.md
* Migration scripts provided when feasible
* Deprecation warnings where possible

.Post-1.0 (Future)
* 6-month deprecation period
* Migration tooling required
* LTS releases for enterprise users
